# Contributing

Thank you for your interest in contributing to this project.

## Scope of Contributions

Contributions are welcome in the following areas:

- code quality improvements
- notebook cleanup and modularization
- documentation improvements
- model evaluation enhancements
- deployment workflow improvements
- bug fixes and repository maintenance

## How to Contribute

1. Fork the repository
2. Create a new branch for your changes
3. Make your updates with clear and consistent commits
4. Test your changes wherever applicable
5. Submit a pull request with a clear explanation of the update

## Contribution Guidelines

- Keep the folder structure organized
- Use meaningful file names
- Write clear commit messages
- Avoid committing unnecessary large files
- Keep documentation professional and consistent
- Preserve reproducibility where possible

## Reporting Issues

If you find a bug or want to suggest an improvement, please open an issue with:

- a short title
- a clear description
- steps to reproduce, if relevant
- expected behavior
- actual behavior

## Notes

This repository is maintained as a project and portfolio showcase. Major structural or documentation improvements are especially appreciated.
